## 关于这个项目要做什么
### 作业管理系统
*** 我们以管理程设 / 计概作业为例（其他的作业比如数分有点太难管理了吧） ***

1. 漂亮地（？）展示当前文件夹中所有作业

- 中间放一个树根，然后外面一圈是各种的作业，然后同一份作业的不同版本会从第一版往后延伸

- 可以做一些美化处理（参考ys的ppt），比如是一棵树或者一个轨道whatever



2. 后端的逻辑不再“抄袭”git，而是依托于我们自己的项目给一些我们能想到的操作

```terminal
ourgit init [your-name] // 初始化一个仓库并输入你的名字

ourgit add [homework-name] [homework-directory] // 将一份新的作业加入我们的管理系统

ourgit commit [homework-name] // 提交某份作业的某个新版本
ourgit commit [homework-directory] // 也可以把参数换成你的作业根目录

ourgit show_difference [homework-name] [commit-id] // 展示某份作业的某个版本相比于上个版本做了哪些改动

ourgit status // 显示当前用户名称、正在完成的作业名称

ourgit history // 显示提交过的所有作业信息和与ai的交互信息

ourgit set_ddl_reminder [homework-name] [time] // 设置作业的ddl提醒
```

- 在这里我能想到的一个点是，可以在这里教用户初步地使用terminal写command line，比如教用户打开terminal，然后走进对应的根目录，然后输入简单的命令。引导式的学习参考。（当然不太好实现）

- 为了方便管理，还可以在人们初始化仓库的时候便将文件夹给做好文件分类管理，比如有.ourGit/, results/, /homework，并提醒用户将对应的作业放在对应的文件夹内，然后提醒他们我们生成的东西在哪里可以看见（这一块做两方面的显示，一方面是把内容放到本地，另一方面是做qt的时候直接显示在用户面前）

3. 调用大模型的一些功能

    (1) 作业批改功能
    - 我们提供两个模式：模式一是单文件批改模式，用户需要输入需要批改的.cpp / .py文件名，并描述他遇到的困难，大模型返回一个.json格式的内容，里面包含修改后的完整版代码，还有修改建议，用户可以选择性选择无脑抄袭或者慢慢学习；模式二是整个项目文件夹的批改模式，比如用户在完成程设大作业的时候遇到了很难debug的地方，就可以调用一下这个模式，我们需要把整个项目给“提取”成一段prompt，大概需要迭代的遍历整个项目文件夹

    (2) 作业启发功能
    - 当用户在做作业的过程中遇到了困难却不想直接得到答案（或者说压根都没做完，就没有所谓地“批改”这一说法），可以使用这个功能，大模型会返回几个启发式的问题引导用户继续完成他的作业。（这个功能其实很实用，比如我自己做ai基础作业的时候就很需要一个这种东西）

    (3) 类似题目的练习功能
    - 可以要求用户将当前不会的作业的题目要求发给ai，让ai生成几个同类型的练习题，放在我们的results/文件夹，然后供用户练习并批改（也可以同时把题目的答案生成出来以供参考）

4. 生成课程总结报告
- 显示整个课程用户完成了几份作业，与ai交互了几次，再调用一下大模型总结一下用户都学到了什么

- 用户一共在我们设计的这个东西上花费了多久
